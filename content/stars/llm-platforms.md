## 최신 트렌딩 (April 2026)

### OpenClaw/openclaw
- **링크:** https://github.com/OpenClaw/openclaw
- **설명:** Open-source personal AI assistant for channels (WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Teams)
- **왜 좋냐:** 2026년 가장 폭발적 성장. 9,000→210,000+ 스타. 자신의 하드웨어에서 실행되는 AI 어시스턴트. Multi-channel 지원.

### getmodo/dify
- **링크:** https://github.com/getmodo/dify
- **설명:** Open-source LLM app development platform with visual workflow, RAG, agents, observability
- **왜 좋냐:** 엔드-투-엔드 LLM 앱 개발 플랫폼. 130,000+ 스타. 시각적 워크플로우 + 프로덕션 준비.

### ollama-ai/ollama
- **링크:** https://github.com/ollama-ai/ollama
- **설명:** Get up and running with Llama 2 and other large language models locally
- **왜 좋냐:** 로컬 LLM 실행의 표준. 162,000+ 스타. macOS, Linux 지원. 모델 관리 간편.

### open-webui/open-webui
- **링크:** https://github.com/open-webui/open-webui
- **설명:** Self-hosted AI platform with offline support, fully compatible with Ollama
- **왜 좋냐:** 자체 호스팅 ChatGPT 대안. 124,000+ 스타. 완전 오프라인 운영 가능. Ollama와 완벽 호환.

### langchain-ai/langchain
- **링크:** https://github.com/langchain-ai/langchain
- **설명:** Building applications with LLMs through composability
- **왜 좋냐:** AI 에이전트 프레임워크의 표준. 에코시스템 가장 큼. Python, JavaScript 모두 지원.

### unstructured-io/unstructured
- **링크:** https://github.com/unstructured-io/unstructured
- **설명:** Open-source library and API to build knowledge bases from raw documents
- **왜 좋냐:** 문서 → 구조화된 데이터 변환. RAG용 문서 전처리 표준. 60,000+ 스타.

### microsoft/MarkItDown
- **링크:** https://github.com/microsoft/markitdown
- **설명:** Python tool for converting files and office documents to Markdown
- **왜 좋냐:** 106,200+ 스타. Microsoft 공식. PDF, Word, Excel 등 → Markdown. RAG 전처리에 유용.
